Object 1003h: Pre-defined Error Field


Sub-index 0:
Scale: NA
Unit: NA
Read: Number of errors in history.
Write: 0 clears error history. Non-zero does nothing.

Sub-index 1-10:
Scale: NA
Unit: NA
Read: Error history entry (see below). The history is arranged chronologically with the most recent error in sub-index 1.

NOTE: If your application uses alarm/fault code mapping, please ignore the following and consult your application-specific documentation for a description of your alarm/fault codes.




Fault Format:
 - Most-Significant 16 bits: 	0xXX00 from CiA301 standard 
	- XX = 90 - External Error - Generic
	- XX = 50 - Hardware Error - Generic
 - Least-Significant 16 bits: 	0xC000h | (DQT Fault ID)
Alarm Format:
 - Most-Significant 16 bits: 	0xXX00 from CiA301 standard 
	- XX = 90 - External Error - Generic
	- XX = 81 - Communications - Generic
	- XX = 82 - Communications - Protocol - Generic
	- XX = 60 - Software - Generic
	- XX = 10 - Generic Error
 - Least-Significant 16 bits: 	0x8000h | (DQT Alarm ID) 
High Priority Alarm Format
 - Most-Significant 16 bits: 	0xXX00 from CiA301 standard 
	- XX = 90 - External Error - Generic
	- XX = 81 - Communications - Generic
	- XX = 82 - Communications - Protocol - Generic
	- XX = 60 - Software - Generic
	- XX = 10 -  Generic Error
 - Least-Significant 16 bits: 	0x0000h | (DQT High Priority Alarm ID) 

NOTE: Due to an idiosyncrasy in Delta-Q's CANopen stack, the 16-bit Delta-Q alarm/fault codes are stored in little-endian order in the EMCY message and big-endian order in the 1003h object (see JIRA reference SSP-4514).

Object 2032h: Can Listen Only Mode


Subindex 0:
Scale: NA
Unit: NA
Read: 
Can listen only mode 
1 = enabled
0 = disabled

Write:
Can listen only mode
1 = enabled, charger CAN controller enters listen-only mode
0 = disabled. charger CAN controller exits listen-only mode

NOTE:
It is advised not to let the charger enter listen-only mode if there will be only one talking  (not listen-only) node on the network after the sdo write action. 

The reason behind is that the charger will not ack messages in this scenario. Unless configured as an one-shot transmitter, the only sender in the CAN network will treat no acking as an error and transmit the message again and again, occupying the bus 100%. A great amount of messages will overflow the receive mail box of the listen-only  charger, and result in message loss for any message sent afterwards

If one puts the charger into listen-only mode in a network, resulting in that only one device is talking (not listen-only) in the network, one may wake up the listen-only charger in two ways:

1. ensure that nothing else is transmitted after the sdo message that puts the charger into listen-only (write 0x01 to sdo obj 0x2032) , then write 0x00 to sdo obj 0x2032 to exit

2. If there were messages transmitted after the sdo message that puts the charger into listen-only (writing 0x01 to sdo obj 0x2032), reset the can bus link and then write 0x00 to sdo obj 0x2032 (this is how the can diagnostic tool wakes up a device in listen-only mode to continue programming the device).

Object 2241h: Active Algorithm Request


Sub-index 0:
Scale: NA
Unit: NA
Read: 
0xFFFFFFFF     No active algorithm is set.
Other Values   
- Bits 0 to 15: The actual active algorithm ID. 
- Bits 16 to 23: The algorithm configuration variant.
- Bits 24 to 31: Reserved, zero

Write: 
- Bits 0 to 16: Algorithm ID.  
- Bits 16 to 23 Algorithm configuration variant.
Ignored if the charger does not support multiple configurations.
If the requested algorithm has configuration variants and the requested variant is installed in the charger then the algorithm is set with the requested configuration variant applied.
Otherwise, if the algorithm has no variants and the requested variant is zero then the algorithm is set with the default configuration. 
Otherwise, the charger responds with an abort.

- Bits 24 to 29: 
Reserved, all these bits should be set to 0

Bits 30 to 31 are a bit field determining the permanence of the selection:

Bit-30:  Temporary Algo AC loss behaviour
   0: The loss of AC will not reset the algorithm back to the permanent algorithm. 
   1: The loss of AC will reset the algorithm back to the permanent algorithm after the AC suspend time has elapsed.  (AC suspend time has a default of 10mins and is the time that the charger allows the AC voltage to be removed before terminating the charge cycle.  The advantage is to prevent short AC Brown-outs restarting charge cycles or resetting temporary algorithms)

Bit-31:  algorithm selection permanence
   0: Select algorithm this time only. (it takes effect until either a charger power down, Battery is removed or in some cases where AC is disconnected depending on Bit-30)
   1:    Store selection permanently.

E.g.  Selecting Algorithm ID 11 permanently:        0x8000000B
      Selecting Algorithm ID 11 non-permanently: 0x0000000B
     Selecting Algorithm ID 28 reset to default algorithm after AC voltage loss for > AC suspend time: 0x4000001C
Selecting Algorithm ID 11 with configuration variant 1 permanently: 0x8001000B

Notes:
1)	If Algorithm ID is not stored in the charger then it will not be set. The current algorithm ID will remain set. 
2)	AC suspend time can be configured by Delta-Q, please contact Delta-Q application engineering to request.
3)	If using the multiple configurations feature (i.e., platform configuration changes with algorithm selection) then the charger will respond with an abort if the permanent bit (Bit-31) is NOT set.
4)	If one or more installed algorithms includes configuration variants then the charger will restart after accepting this request. Object 6080h: Charger State of Charge


Sub-index 0: 
Scale: Q0
Unit: Percent
Read: 0 if the charger is indicating less than 80% (see object 0x2006) and the battery voltage is less than the zero state-of-charge voltage, 50 if the charger is indicating less than 80% (see object 0x2006) and the battery voltage is greater than the zero state-of-charge voltage, 81 if the charger is indicating greater than 80%, 90 if the charger is indicating finishing and 100 if the charger is indicating complete. Default value if measurements are not available. Previous value if resting, equalizing or charging terminated by algorithm.

See CiA 419 Device Profile for Battery Chargers for more information.
